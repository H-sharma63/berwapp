# Contribution Guide

Thanks for your interest in improving `berwapp`! We welcome all contributions.

## 🛠 How You Can Help

- 🐛 **Report bugs** — Open an issue with reproduction steps
- 🚀 **Suggest features** — Use the feature request template
- 💻 **Write code** — Pick an issue tagged `good first issue` or `help wanted`
- 📖 **Improve docs** — Clarify guides, fix typos, add examples
- 🌐 **Translate** — Help make `berwapp` accessible in other languages

## 🔄 Workflow

1. **Fork & clone:**
    ```bash
    git clone https://github.com/H-sharma63/berwapp.git
    ```

2. **Create a branch:**
    ```bash
    git checkout -b feat/your-feature
    ```

3. **Install deps:**
    ```bash
    npm install
    ```

4. **Develop:**
    ```bash
    npm run dev  # hot reload
    ```

5. **Test:**
    ```bash
    npm test
    ```

6. **Build:**
    ```bash
    npm run build
    ```

7. **Commit & push:**
    ```bash
    git commit -m "feat: add your feature"
    git push
    ```

8. **Open a PR** on GitHub.

---

## 📜 Guidelines

- Code style → Auto-formatted via Prettier
- Commit messages → Use conventional commits (`feat:`, `fix:`, `docs:`)
- PRs → Link to an open issue when possible
- Code of Conduct → Be respectful and constructive
